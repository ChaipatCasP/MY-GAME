# Milestone 1: MVP Prototype

**Target Date**: 2026-04-25 (4 weeks from project start)
**Status**: Active
**Owner**: Solo Developer

## Goal

A playable mobile prototype that validates the core question:
> "Is the plant → water → harvest → deliver loop satisfying in a 15-minute mobile session?"

## Success Criteria

- [ ] Touch tile grid is playable on a physical device (or Godot mobile sim)
- [ ] At least 1 spirit quest can be completed (plant X, deliver to spirit)
- [ ] The day cycle advances naturally across a 15-minute session
- [ ] Save/load works reliably (app can be backgrounded and resumed)
- [ ] Demo-able without a tutorial (visual clarity alone guides the player)

## Scope (MVP Systems Only)

From systems-index.md — MVP tier:
1. Crop & Seed Database
2. Spirit & Island Data
3. In-Game Day Cycle
4. Farm Grid Manager
5. Inventory / Crop Storage
6. Tile Farming System
7. Spirit Quest System
8. Save / Load System
9. Farm HUD
10. Quest Tracker UI

## Out of Scope for This Milestone

- Drift / Travel System (Vertical Slice milestone)
- Island Transition Cinematic
- Seed Discovery System
- Real pixel art (placeholder OK)
- Audio (placeholder OK)
- Multiple islands

## Sprints

- Sprint 1 (Week 1): Prototype validation + Foundation systems design
- Sprint 2 (Week 2): Core systems design + Farm Grid implementation
- Sprint 3 (Week 3): Quest system + Save/Load implementation
- Sprint 4 (Week 4): UI + integration + device testing

## Risks

| Risk | Mitigation |
|------|-----------|
| Touch controls feel bad | Sprint 1 prototype validates this immediately |
| Save/load unreliable on mobile | Prototype save pattern early in Sprint 3 |
| Scope creep | Refer to systems-index.md — nothing outside MVP tier |
